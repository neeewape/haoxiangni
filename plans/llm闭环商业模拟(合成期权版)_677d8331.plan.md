# 红枣“月1%”商业模拟系统：合成期权近似版闭环计划（基于两份CSV）

## 你当前提供的数据

- **期货**：`红枣期货数据.CSV`
- 字段（从文件内容推断）：日期 + 开高低收 + 成交量（表头编码乱码，但数据结构清晰）
- 频率：日频
- **期权**：`红枣期权数据.CSV`
- 表头：`交易日期,日均隐含波动率(%),日最低隐含波动率(%),日最高隐含波动率(%),日波动率标准差(%),日合约数量,成交量(手)`
- 频率：日频
- 备注：**无到期/行权价/认购认沽/期权价格**，因此只能做“合成期权/参数化IV”近似，不做真实合约链回测。

## 最小实验目标（不搞复杂）

- **验证**：在“支撑/压力附近双卖 + 期货对冲 + 保留10%敞口”规则下，是否有参数区间能让**月度收益中位数≥1%**，并给出**最大回撤分布**与建议头寸规模。

## 策略与数据缺口的处理方式（你已选择）

- **支撑/压力与方向信号缺历史**：用期货价格自身生成代理信号（先跑起来，再迭代更贴近现货团队的版本）。
- 支撑/压力代理：滚动分位数/布林带/ATR通道（三选一，LLM可迭代选择）
- 方向代理：均线趋势/动量/突破（涨/跌/震荡三分类）

## 实验引擎（合成期权蒙特卡洛）

### 1) 价格过程（短期随机游走，可增强但保持简单）

- 用期货日收益拟合短期分布：
- 基线：正态/学生t（更适合厚尾）
- 可选：简单GARCH捕捉波动聚集（仅当厚尾/聚集明显影响结果时才加）

### 2) 波动率过程（用你给的IV统计驱动）

- 每个交易日从 `日均IV` 与 `日波动率标准差` 构造IV水平；
- 用 `日最低IV/日最高IV` 作为**波动率区间约束**做压力情景；
- 合成“1个月到期的ATM IV”作为期权定价输入（近似）。

### 3) 合成期权定价与双卖结构

- 用 Black-76（期货期权常用）对“给定到期T、行权价K、期货价F、IV、利率r”定价合成Call/Put。
- 双卖规则（参数化，供LLM迭代）：
- 以代理支撑/压力为锚设定K_put、K_call（如：K_put = support*(1-α)，K_call = resistance*(1+β) 或直接取支撑/压力）
- 合约期限固定近似1个月（20-23交易日滚动）

### 4) 期货对冲 + 10%敞口

- 用合成Greeks（Black-76 Delta）做对冲：
- 目标净Delta = 方向信号 * 10% * 名义规模
- 其余Delta用期货对冲
- 输出对冲交易次数与交易成本敏感性（用滑点/手续费参数化）。

### 5) 行权/接货优势的近似表达（因缺真实合约条款/现货成本数据）

- 在合成模型中用“尾部止损/展期/转仓”规则近似你们接货处理能力；
- 报告中明确：**此处为近似占位**，若未来补充仓储/资金/基差数据可替换为真实接货PnL路径。

## “验证前四项是否成立”的落地方式（在现有数据范围内）

1. **短期随机游走**：用期货日收益做 ACF/Ljung-Box、方差比等检验。
2. **长期供需决定**：当前无供需因子数据，只能做弱验证：长期趋势/结构变化检测（分段均值/波动/断点）；若你后续提供供需数据再做因子解释。
3. **现货团队方向90%正确率**：当前无团队历史信号，暂不能验证；先用代理信号替代，并把该项标记为“待补数据验证”。
4. **策略能否月1%**：用蒙特卡洛/历史回放（基于拟合分布）输出收益与回撤分布，并做压力情景。

## LLM闭环迭代（简单停机条件）

- 目标函数：月度收益中位数（或P40）- 回撤惩罚 - 爆仓/极端亏损惩罚
- 达标停机（默认，可调）：
- 月度收益中位数 ≥ 1
